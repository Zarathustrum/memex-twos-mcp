# Contributing to Memex Twos MCP

Thanks for your interest in improving this project.

## Reporting Bugs

- Use the bug report template in GitHub issues.
- Include steps to reproduce and expected vs actual behavior.
- Redact any personal data from examples.

## Suggesting Features

- Use the feature request template.
- Explain the user benefit and any trade-offs.

## Pull Requests

1. Fork the repo and create a feature branch.
2. Add or update tests when applicable.
3. Keep changes small and focused.
4. Ensure formatting and tests pass.

## Code Style

- Follow PEP 8.
- Add type hints for public functions.
- Write docstrings for modules, classes, and public functions.
- Prefer clear, readable implementations over clever ones.

## Privacy

- Do not commit personal Twos data.
- Avoid hardcoded paths and user-specific details.
- Use sample data for tests and examples.
